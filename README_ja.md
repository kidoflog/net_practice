### Terms

#### TCP/IP addressing
- TCP/IP は，コンピュータネットワークにおいて，世界標準的に利用されている通信プロトコルです．TCPは正確にデータをやり取りするために定められた通信プロトコルです．一方，IPとはIPアドレスと呼ばれる数値を用いて通信先の指定を行うためのプロトコルです．IPアドレスにはIPv4とIPv6が存在しますが，現在主流であり，この課題で用いるのは32bitのIPv4です．IP addressingは，このIPアドレスを用いて通信先を一意に特定することを指します．
#### subnet masks
- サブネットとは，大きなネットワークを分割した，小さなネットワーク単位のことです．サブネットを用いる事で，通信が届く範囲を限定し，無駄な通信を減らすことでルーティングが効率的に行えるようになります．ユーザーはサブネットマスクを用いることで，IPアドレスをネットワーク部とホスト部に分割します．サブネット内のホストは同一のネットワーク部を持つことにより，サブネットを構成する事ができます．
#### default gateways
- デフォルトゲートウェイは，ネットワーク内からのデータが外部へ送信される際に使われるネットワーク機器のことを指し，一般にはルーターのことを指します．送信先が同じネットワーク内に存在しない場合に利用され，インターネットにアクセスするための出入り口となります．
#### routers and switches
- ルーターは，前述したデフォルトゲートウェイに用いられる機器で，データが適切に送受信されるように，ルートを規定する(ルーティング)役割を持ちます．
- スイッチは，同じネットワーク内に存在する機器にデータを送受信するための中継機器です．
#### OSI layers
- OSI参照モデルとは，コンピュータネットワークに求められる機能を7段層の構造に分割し，定義したものです．
- アプリケーション層は，ユーザが操作するソフトウェアの具体的な機能や通信手順，データ形式などを定めています．これは使用するアプリケーションによって無数に存在するため，アプリケーションに応じてハードウェアの通信機能を用います．
- プレゼンテーション層では，データを適切なフォーマットにすることにより，アプリケーション層に適切なデータを渡す事ができます．
- セッション層では，ホスト間の通信の確率，維持，終了の手順を規定しています．現在主流となっているプロトコルはTLSです．TLSは，電子証明書を用いることにより，安全な通信を行うことを可能とします．
- トランスポート層は，信頼性やリアルタイム性などの通信品質を実現するための層で，代表的なのはTCP,UDPです．TCPは信頼性，UDPはリアルタイム性の面で優れたプロトコルです
- ネットワーク層では，複数のネットワークを繋いで，相互に通信することを実現する層で，代表的なプロトコルはIPです．
- データリンク層では，直接接続された機器(ノード)同士の通信を実現します．代表的なものは，MACアドレスを用いたイーサネット接続です．MACアドレスとは，ノード間のデータ転送を行う際の宛先として利用されます．
- 物理層は，ハードウェアのコネクタ形状等，物理的な接続を定めるものです．
